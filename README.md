# 家計簿ライフプラン — セットアップ手順

このアプリは **そのまま index.html を開くだけ** で全機能（ライフプラン・家計簿・グラフ）が使えます。
スマホとPCでデータを共有したいときだけ、下の手順で「公開」と「同期」を設定してください。

---

## 手順1: Supabaseの準備（無料・約10分）

1. https://supabase.com にアクセスし、無料アカウントを作成
2. 「New project」でプロジェクトを作成（リージョンは Tokyo 推奨、DBパスワードは控えておく）
3. 左メニューの **SQL Editor** を開き、同梱の `supabase-schema.sql` の中身を貼り付けて **Run**
   - 「Success. No rows returned」と出ればOK（plans・entriesテーブルが作られます）
4. 接続情報を2つコピーする
   - **Project URL**（`https://xxxx.supabase.co`）… 左メニュー **Data API** ページに表示
   - **公開キー** … 左メニュー **Project Settings → API Keys** に表示。次のどちらでも動きます
     - 新方式: **Publishable key**（`sb_publishable_...`）← 推奨。そのまま使えます
     - 旧方式: **Legacy anon, service_role** タブの **anon public**（`eyJ...`）

※ この公開キーは「ブラウザに置いて公開してよい」種類のキーです。データ保護はRLS（行レベル
セキュリティ）が担っており、schema.sqlでログイン本人の行しか読み書きできないよう設定済みです。
（secret / service_role キーは絶対に貼らないでください）

## 手順2: アプリを公開する（GitHub Pages・無料）

スマホからアクセスするには、このフォルダをWebに公開する必要があります。

1. https://github.com で新しいリポジトリを作成
   - **無料アカウントでは「Public（公開）」を選ぶ必要があります**（PrivateのPagesは有料プラン限定）
   - 公開されるのはアプリのコードのみ。家計データやSupabaseキーはコードに含まれないため安全です
2. このフォルダのファイル一式（index.html, sw.js, manifest.webmanifest, icon.svg, README.md, supabase-schema.sql）をアップロード
3. リポジトリの **Settings → Pages** → Branch を `main` にして Save
4. 数分後に `https://ユーザー名.github.io/リポジトリ名/` でアプリが開けます

※ URLを知っている人はアプリ自体を開けますが、データはログインしないと見えません（RLSで本人の行のみ保護）。

## 手順3: アプリで同期を設定

1. 公開URLをPCとスマホの両方で開く
2. **設定タブ** → Supabase URLとanon publicキーを貼り付け → 「接続設定を保存」
3. メールアドレスとパスワードを入れて「新規登録」（初回のみ。確認メールが届いたらリンクを開く）
4. PC・スマホそれぞれで「ログイン」→「今すぐ同期」

以降は入力のたびに数秒後、自動で同期されます。

## 手順4: スマホでアプリ化（任意）

- **iPhone**: Safariで開く → 共有ボタン → 「ホーム画面に追加」
- **Android**: Chromeで開く → メニュー → 「アプリをインストール」

---

## データの保存場所まとめ

| 状態 | 保存場所 |
|---|---|
| 同期未設定 | この端末のブラウザ内のみ |
| 同期設定済み | 端末内 + Supabase（自分のアカウントの行のみ） |
| エクスポート | JSONファイル（ライフプラン+家計簿の完全バックアップ） |

## トラブルシューティング

- **「同期エラー: Failed to fetch」** → URL/キーの貼り間違い、またはオフライン
- **「未ログインのため同期できません」** → 設定タブでログイン
- **新規登録後にログインできない** → 確認メールのリンクを先に開く
- **file://で開くと保存されない警告が出る** → その端末ではエクスポート/インポート運用にするか、公開URL経由で使う
